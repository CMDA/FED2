Frontend Development V2

API

An application-programming interface (API) is a set of programming instructions and standards for accessing a Web-based software application or Web tool. A software company releases its API to the public so that other software developers can design products that are powered by its service.

Frontend Development V2

REST API

  • ReST = Representational State Transfer
  • Uses HTTP protocol to make calls between machines
  • REST is an architecture style for designing networked applications
  • HTTP requests are used to read, post and update data

Frontend Development V2

REST API - CRUD

  • CRUD = Create/Read/Update/Delete
  • Read uses GET request
  • Other use POST request

Frontend Development V2

JSON

  • JSON = JavaScript Object Notation
  • JSON is a syntax for passing around objects that contain name/value pairs, arrays and other objects
  • Parsing JSON is much easier and quicker than XML, just 'eval' the JSON
  • JSON.parse(data) & JSON.stringify(data)

Frontend Development V2

AJAX

  • AJAX = Asynchronous JavaScript and XML
  • Uses XMLHttpRequest (XHR)
  • Data-driven as opposed to page-driven
  • AJAX is a web browser technology independent of web server software
